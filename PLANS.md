# ExecPlan

## 任务
- 将 `心理危机干预与预防/Notes/` 下 7 份 Markdown 课堂笔记合并排版为一份纸质开卷考试用 PDF。
- PDF 前置可打印目录，目录条目必须带真实页码，不能只依赖电子超链接。
- PDF 每页需要页眉与页脚：页眉显示课程/章节信息，页脚显示页码。
- 排版目标是“考试速查优先”：压缩字体、行距、段距和列表间距，同时保持中文清晰可读。

## 目标拆解
1. 核对本机工具链：
   - 优先使用 Pandoc；
   - 中文 PDF 使用 XeLaTeX；
   - 若 `xelatex` 不在 PATH，则显式调用 `D:/App/TinyTeX/bin/windows/xelatex.exe`。
2. 读取 7 份笔记标题结构，整理适合考试速查的目录层级：
   - 保留每份笔记的主标题；
   - 保留二级标题作为主要检索入口；
   - 选取关键三级/四级标题进入详细目录，避免无意义短标题把目录冲散。
3. 生成可复现的导出工程：
   - 合并 Markdown；
   - 准备 Pandoc 元数据、XeLaTeX header、必要的 Lua 过滤器或脚本；
   - 控制字体、字号、页边距、行距、段距、列表间距、页眉页脚和目录深度。
4. 编译 PDF：
   - 至少运行到目录页码稳定；
   - 失败时优先修复中文字体、LaTeX 包、特殊字符、标题层级或 Markdown 语法问题。
5. 验证输出：
   - 用 PDF 文本提取检查目录确实带页码；
   - 渲染首页、目录页、正文页和末页为图片，检查中文、页眉、页脚、页码、密度和换页；
   - 统计总页数，确认没有异常空白页或明显过度膨胀。
6. 更新 `PROGRESS.md`，提交本轮产物和脚本，并向用户说明最终 PDF 路径与验证结果。

## 执行约束
- 不改动原始 7 份课堂笔记正文，除非发现会阻断编译的语法问题且必须修复。
- 生成文件集中放在 `output/pdf/心理危机干预与预防/`，避免污染课程原始笔记目录。
- 目录以纸质检索为核心：必须显示页码；允许保留 PDF 内部跳转，但不能把跳转当成唯一目录。
- 由于仓库已有大量无关未提交文件，本轮提交只暂存与本任务直接相关的文件。

## 常见坑 / 复现方法
- 坑1：旧脚本会跳过“先说我觉得最重要的机制问题”这类前导段落。（复现：运行第一幕旧脚本后查看问题 4 之前的导语，页面中不会出现。）
- 坑2：旧脚本把总结区标题写死成“总结第一幕分析结论”，无法适配“本幕总结（当前阶段结论）”。（复现：将第二幕 Markdown 直接交给旧脚本运行。）
- 坑3：Edge 无头打印直接输出到中文 PDF 文件名时，可能不报错但不落盘。（复现：将 `--print-to-pdf` 指向 `Case1-第二幕-美化版.pdf` 这类中文文件名。）
- 坑4：单独依赖 Edge 命令行 `--print-to-pdf` 不够稳定，而 Playwright 的 `page.pdf()` 更可靠。（复现：分别测试两种导出方式，对比落盘成功率。）
- 坑5：若“关键词”后面先跟 `---` 再跟导语，不能按“直到下一个同级标题”为止直接截关键词，否则导语会被误并入关键词。（复现：第二幕与第一幕的关键词区都能复现。）
- 坑6：如果某个问题块正文最后正好以 `---` 结束，再叠加 HTML 里的 `.qa-section` 顶部分隔，就会出现双重分隔。（复现：问题 1/2、2/3 等相邻问题边界都能看到。）
- 坑7：单页 HTML 默认是“深色页面背景 + 居中缩放 + 白色画布阴影”，如果直接截图，会把黑边或透明边带进 PPT。（复现：不用 `?export=1` 直接对 `PPT-问题4-2.html` 等页面做 1920×1080 截图。）
- 坑8：如果旧版支持页或总稿仍在 WPS / VS Code 中打开，直接覆盖原文件名会失败。（复现：打开 `总PPT构建/output/*.pptx` 后重新运行 `build_all.ps1`；当前脚本已自动改为落到 `-修订版N` 文件名。）
- 坑9：自制 HTML 页在浏览器整窗截图时，底部页脚不一定能完整进入最终 PNG，因此不要为了补页脚去改共享 CSS；更稳的做法是在 `build_support_deck.js` 的图片页阶段把页脚叠回 PPT。（复现：对 `PPT-问题5.html` 做 1920×1080 整窗截图，再和 1250 高窗口截图对比，可见页脚处于视口边缘。）
- 坑10：老师临时补内容时，Word 里往往是“蓝字标题 + 红字要点”，不适合整段搬进 PPT；更稳的方式是用 `python-docx` 先抽出着色文字，再整理成课堂讨论提示页。（复现：读取 `新建 DOCX 文档.docx`，可见“病因 / 分子机制 / 诱因及高危因素 / 鉴别诊断”等均以颜色标注。）
